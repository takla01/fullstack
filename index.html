Section 01

What is Full Stack Development?

Full stack development refers to building both the front-end (client side) and back-end (server side) of a web application — the complete picture from the user's screen to the database.

🖥️
Front End — Presentation Layer
HTML · CSS · JavaScript · React / Vue / Angular
⚙️
Back End — Logic & API Layer
Node.js · Python · PHP · REST / GraphQL APIs
🗄️
Database — Persistence Layer
MySQL · PostgreSQL · MongoDB · Redis

The Client–Server Model

When you visit a website, your browser (client) sends an HTTP request to a web server. The server runs business logic, queries a database, then returns data — often as JSON — which the front end renders into the UI you see. A full stack developer understands and can work across every layer of this cycle.

Why Learn Full Stack?

A full stack developer can design a database schema, build a REST API, wire up authentication, and produce a polished, accessible UI — all within a single project. This makes them exceptionally versatile in small teams and highly valuable in larger ones.

Common Stacks

MEAN MERN LAMP JAMstack Django + React Rails + Vue — each pairing specific front-end and back-end technologies chosen for the project's needs.

Section 02

HTML

HyperText Markup Language is the skeleton of every web page. It provides the semantic structure that browsers render and assistive technologies interpret.

semantic-page.html 
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>My Web Page</title>
</head>
<body>

  <header>                         <!-- site-wide header -->
    <nav></nav>
  </header>

  <main>                          <!-- primary content -->
    <article>
      <h1>Page Heading</h1>
      <p>Paragraph of content.</p>
      <img src="photo.jpg" alt="Descriptive alt text" />
      <a href="/about">Learn more</a>
    </article>

    <aside>Related links</aside>
  </main>

  <footer>© 2025</footer>

</body>
</html>

The code above demonstrates semantic HTML5 — using meaningful elements (<header>, <main>, <article>, <aside>, <footer>) instead of generic <div> wrappers. This improves accessibility, SEO, and maintainability.

Block vs Inline

Block elements (<div>, <p>, <h1>) take full width; inline elements (<span>, <a>, <strong>) sit within text flow.

Attributes

Attributes provide extra information: href on links, src on images, alt for accessibility, id and class for targeting with CSS/JS.

Accessibility

Semantic elements give screen readers context. Always use alt on images, <label> on inputs, and logical heading hierarchy.

Section 03

CSS

Cascading Style Sheets control every visual aspect of a web page — layout, colour, typography, spacing, and animation — through a cascade of rules applied to HTML elements.

styles.css — key concepts 
/* Selector → Declaration Block */
h1 {
  font-size: 2rem;
  color: #0A0F1E;
  font-family: 'Space Grotesk', sans-serif;
}

/* CSS Custom Properties (variables) */
:root {
  --primary-color: #2563EB;
  --border-radius: 8px;
}

/* CSS Grid — 3-column responsive layout */
.grid-container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 1rem;
}

/* Flexbox — horizontal centering */
.flex-center {
  display: flex;
  justify-content: center;
  align-items: center;
}

/* Keyframe animation */
@keyframes pulse {
  0%, 100% { transform: scale(1);   }
  50%        { transform: scale(1.1); }
}

/* Media query — mobile-first responsive design */
@media (max-width: 640px) {
  .grid-container {
    grid-template-columns: 1fr;
  }
}

The Box Model

Every HTML element is a box with four layers: content → padding → border → margin. Using box-sizing: border-box makes width calculations include padding and border, preventing unexpected overflow.

Specificity & the Cascade

When multiple rules target the same element, the browser resolves conflicts via specificity (inline > IDs > classes > elements) and source order. CSS Custom Properties (--var) let you define a design token once and reuse it everywhere.

Section 05

JavaScript

JavaScript is the language of the web. It runs in the browser, manipulates the DOM, handles events, fetches data from APIs, and powers everything from simple interactions to full-stack applications via Node.js.

core-js.js — key concepts 
// Variables & Data Types
const name   = "Alice";       // string (immutable binding)
let   score  = 0;             // number (mutable)
const skills = ["HTML", "CSS", "JS"]; // array

// Functions (arrow syntax)
const greet = (name) => `Hello, ${name}!`;

// DOM Manipulation
const btn = document.querySelector("#myButton");
btn.addEventListener("click", () => {
  document.querySelector("#output").textContent = greet(name);
});

// Async – Fetch API (GET request)
async function getUser(id) {
  const res  = await fetch(`/api/users/${id}`);
  const data = await res.json();
  return data;
}

// Array Methods
const nums    = [1, 2, 3, 4, 5];
const doubled = nums.map(n => n * 2);          // [2,4,6,8,10]
const evens   = nums.filter(n => n % 2 === 0); // [2,4]
const total   = nums.reduce((acc, n) => acc + n, 0); // 15

    The DOM

    The Document Object Model is the browser's in-memory tree representation of the page. JavaScript can read, create, modify, or delete any node, making pages fully dynamic.

    Event Loop

    JavaScript is single-threaded but non-blocking. The event loop processes a task queue so async operations (timers, fetch calls, user events) don't freeze the UI.

    Modern JS (ES6+)

    Arrow functions, destructuring, template literals, async/await, and modules (ES6+) make JavaScript expressive and maintainable at scale.
